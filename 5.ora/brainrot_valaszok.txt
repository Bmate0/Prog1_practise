03 Melyik Boss Karakter jelenik meg? — Hibák azonosítása
a) int* p = nullptr; printf("%d", *p);
Boss: Segfault Samu (vagy Null Nick hibás használata).
Miért: Mert egy null mutatót (nullptr) próbálunk dereferenciálni (kiolvasni az értékét), ami érvénytelen memóriahozzáférés.
b) int* p = (int*)malloc(sizeof(int)); *p = 42; // ... free() soha nem hívódik
Boss: Memory Leak Miki
Miért: Mert foglaltunk memóriát a heap-en, de nem szabadítottuk fel, így az a memória "elveszett" a rendszer számára.
c) int vegtelen(int n) { return vegtelen(n); }
Boss: Stack Overflow Stan
Miért: Mert a függvény végtelenül hívja önmagát (Recursion Rick elfelejtette a base case-t), és a call stack (verem) betelik.

04 Saját Brainrot Karakter — Rövid esszé
Név: Const Connie
Fogalom: const módosító
Személyiség: Connie egy rendkívül makacs és konzervatív karakter. Ha egyszer kijelent valamit, onnantól kezdve senki nem veheti rá a változtatásra. Biztonsági őrnek képzeli magát, aki megvédi az értékeket a véletlen átírástól. Ha megpróbálod megváltoztatni a véleményét, azonnal panaszt tesz a Fordítónál.
Idézet: "Megmondtam, hogy nem! Amit egyszer leírtam, az kőbe van vésve."
Kódpélda:
const int max_elet = 100;
max_elet = 110; // ✗ HIBA: Connie nem engedi a változtatást!
